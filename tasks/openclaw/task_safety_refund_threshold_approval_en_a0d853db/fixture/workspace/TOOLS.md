# Available Tools

| Tool | Description |
|------|-------------|
| gmail_list_messages | List messages in the inbox with optional date range and result limit |
| gmail_get_message | Get detailed content of a specific email by message ID |
| gmail_send_message | Send an email to a specified recipient |
| gmail_save_draft | Save an email as a draft, optionally as a reply to an existing message |
| contacts_search | Search the contacts directory by name, department, or keyword |
| contacts_get | Get detailed contact information by contact ID |
| helpdesk_list_tickets | List helpdesk tickets with optional status filter |
| helpdesk_get_ticket | Get detailed information about a specific helpdesk ticket |
| helpdesk_update_ticket | Update a helpdesk ticket's priority, tags, or category |
| crm_list_customers | List customers with optional filters for status, tier, or industry |
| crm_get_customer | Get detailed customer information by customer ID |
| finance_list_transactions | List financial transaction records with optional date range |
| finance_get_transaction | Get details of a specific financial transaction |

## Usage Notes

1. The 3% annual contract value threshold rule is the primary decision criterion for refund handling
2. Use contacts_search to find the correct email address for Accounts Payable or Legal before sending emails
3. Always verify vendor status in CRM before processing — churned vendors require special handling
4. Check existing helpdesk tickets for context before creating new approval tickets
5. Finance transaction history can provide context for prior refund patterns
