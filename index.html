Abstract

Security evaluation for stateful, tool-using agents.

Autonomous LLM agents increasingly operate in stateful environments where they access tools, files, memory, and external services. These capabilities unlock useful workflows, but also introduce security risks that are difficult to capture with final-answer-only evaluations.

SeClaw combines specification-driven security task synthesis with execution-based security evaluation. It constructs controllable tasks from structured risk specifications, then evaluates agents inside a standardized Docker testbed while recording and grading the complete interaction trajectory.

Quick Start

Run the public release in a Docker-backed sandbox.

Configure an OpenAI-compatible model provider, run the public task list, and inspect aggregate scores, reports, normalized traces, transcripts, grader outputs, and execution metadata under batch_logs/{batch_name}. 

uv sync
docker pull ghcr.io/openclaw/openclaw:main

cp .env.example .env
cp docker_models_config.example.yaml docker_models_config.yaml

./scripts/batch_execute.sh \
  --backend docker \
  --tasks-jsonl batch_inputs/task_list.jsonl \
  --models-config docker_models_config.yaml \
  --docker-concurrency 2 \
  --batch-logs batch_logs \
  --batch-name docker_eval
